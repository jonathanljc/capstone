# PI-HLNN V2: Input-Conditioned Tau Architecture

## What Changed from V1

| Component | V1 | V2 |
|-----------|----|----|
| h0 (initial state) | FP-seeded (0.01 × tanh) | Zeros |
| cm, gleak | Fixed learned scalars | Derived from [F1,F2,F3,CIR_peak] × 0.01 |
| Features | F1, F2, F3 (3 features) | F1, F2, F3, CIR_peak (4 features) |
| Dual circuits | Yes | Yes (unchanged) |
| Cross-circuit talk | Yes | Yes (unchanged) |
| Tau separation loss | Yes | Yes (unchanged) |

## Architecture Flow — Step by Step

### Step 1: Derive ODE Parameters from Physical Features

```
[F1, F2, F3, CIR_peak] × 0.01
        │
        ├──→ cm_net_los(4→32)    → softplus → cm_los     (32-dim)
        ├──→ gleak_net_los(4→32) → softplus → gleak_los  (32-dim)
        ├──→ cm_net_nlos(4→32)   → softplus → cm_nlos    (32-dim)
        └──→ gleak_net_nlos(4→32)→ softplus → gleak_nlos (32-dim)
```

Each circuit gets its OWN cm/gleak tuned to this sample's signal characteristics.

- **LOS sample** (high F1, CIR_peak ≈ F1): one set of cm/gleak → one tau regime
- **NLOS sample** (low F1, CIR_peak >> F1): different cm/gleak → different tau regime

This is the V2 change — tau is now a **direct function of the physical signal features**.

### Step 2: Initialize Hidden States

```
h_los  = zeros (32-dim)
h_nlos = zeros (32-dim)
```

No FP seeding — FP features enter only through cm/gleak (Step 1).

### Step 3: Process CIR Sequence (t = 0 to 59)

For each timestep:

**3a. Cross-Circuit Talk**
```
proj_nlos→los = P_nlos2los(h_nlos)          # project NLOS state into LOS space
proj_los→nlos = P_los2nlos(h_los)           # project LOS state into NLOS space

gate_los  = sigmoid([h_los  | proj_nlos→los])   # how much to let NLOS influence LOS
gate_nlos = sigmoid([h_nlos | proj_los→nlos])   # how much to let LOS influence NLOS

h_los_in  = h_los  + gate_los  × proj_nlos→los  # LOS state + gated NLOS info
h_nlos_in = h_nlos + gate_nlos × proj_los→nlos   # NLOS state + gated LOS info
```

**3b. ODE Step (each circuit independently)**
```
LOS circuit:
  Uses cm_los, gleak_los from Step 1
  ODE: v = (cm_t × v + gleak_los × vleak + w_num + sensory_current) / (cm_t + gleak_los + w_den)
  tau_los = cm_los / (gleak_los + w_den)
  h_los = clamp(v, -1, 1)

NLOS circuit:
  Uses cm_nlos, gleak_nlos from Step 1
  ODE: v = (cm_t × v + gleak_nlos × vleak + w_num + sensory_current) / (cm_t + gleak_nlos + w_den)
  tau_nlos = cm_nlos / (gleak_nlos + w_den)
  h_nlos = clamp(v, -1, 1)
```

Save h_los[t], h_nlos[t], tau_los[t], tau_nlos[t] at each timestep.

### Step 4: Pool and Fuse

```
h_los_avg  = mean(h_los[0..59])   → 32-dim
h_nlos_avg = mean(h_nlos[0..59])  → 32-dim
h_fused    = [h_los_avg | h_nlos_avg] → 64-dim
```

Time-averaged ODE trajectory — no attention, no learnable pooling.

### Step 5: Classify

```
64 → Linear(64,32) → SiLU → Dropout → Linear(32,1) → Sigmoid → P(NLOS)
```

## How the Circuits Specialize (LOS vs NLOS)

The two circuits are architecturally identical — nothing hardcodes which is "LOS" or "NLOS". Specialization comes from the **tau separation loss** during training:

```
Δτ = tau_nlos_circuit - tau_los_circuit

LOS input  (label=0): loss wants Δτ > +margin  (NLOS circuit calm, LOS circuit reacts)
NLOS input (label=1): loss wants Δτ < -margin  (LOS circuit calm, NLOS circuit reacts)
```

This hinge loss forces the two circuits to develop **opposite tau responses** to LOS vs NLOS signals. Without it, both circuits would collapse to identical dynamics.

## Loss Function

```
loss = BCE(smooth_labels) + λ_sep × tau_separation_loss
```

- **Label smoothing** (0.05): prevents overconfident predictions
- **Tau separation** (λ=0.10, margin=0.10): forces circuit specialization

## Why CIR Peak (NLOS Peak)?

F1-F3 only describe the first-path arrival. CIR peak captures the **strongest path** in the entire impulse response.

| Sample | F1 | CIR Peak | Peak/F1 | Meaning |
|--------|-----|----------|---------|---------|
| LOS | High | High | ≈ 1.0 | First path IS the strongest |
| NLOS | Low | High | >> 1.0 | A reflected bounce is stronger than direct path |

This ratio is a classic NLOS indicator — now wired directly into the ODE via cm/gleak.

## Why This is a Liquid Neural Network

The core of an LTC (Liquid Time-Constant) network is the ODE:

```
dh/dt = -h/τ + f(h, x)
```

where τ (tau) is the time constant controlling how fast the system responds. In V2, τ is **fully input-conditioned**:

```
τ = cm(F1,F2,F3,peak) / (gleak(F1,F2,F3,peak) + w_den(h))
```

- `cm` depends on input features (via cm_net)
- `gleak` depends on input features (via gleak_net)
- `w_den` depends on hidden state (state-dependent)

This is closer to the original LTC paper (Hasani et al. 2020) where time constants adapt to the input.
